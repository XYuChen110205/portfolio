## 阶段五：设计语言调整

### 5.1 Emoji → SVG 矢量图标

**做了什么**：创建 `js/icons.js`（40个Feather风格SVG图标），替换全站所有emoji

**为什么这么做**：
- Emoji在不同操作系统显示效果不一致（Win/Mac/手机样式各异）→ SVG是真正的矢量，像素级一致
- Emoji "花里胡哨"（用户原话）降低了专业感 → SVG简洁克制，Stroke线条风
- SVG可以通过CSS `color` 统一配色，保持设计一致性

**替换范围**：
- 21个项目卡片图标
- 详情面板8个分类标题图标
- 6个技能图标 + 3个联络图标
- Footer爱心 → SVG心形

### 5.2 Complete.dev 风格尝试

**做了什么**：制作了 `demo-complete-style.html` 对比demo（Inter字体+清爽商务风）

**为什么放弃**：用户对比后决定保留原有"粒子背景+暗色炫酷"方向，只说emoji太花

### 5.3 顶部导航 → 侧边栏

## 阶段七：中英双语系统

### 做了什么
1. 新建 `js/i18n.js` — 包含所有文案的中英对照表（zh/en 两套）
2. 侧边栏添加 `中 / EN` 切换按钮
3. 所有文案（导航/标题/描述/项目简介/技能/时间线/联系）使用 `t('key')` 翻译函数

### 为什么这么做
- 用户直接说"可以做双语吗" → 中英双语是个人网站加分项
- 英文版让GitHub/海外访客也能理解项目内容
- 只翻译了UI文案和项目简介，架构图内的中文标签保持原样（图表是技术文档，翻译会破坏结构）

### 实现方式
- `i18nMap = { zh: {...}, en: {...} }` 对照表
- `t(key)` → 查表返回当前语言文本
- `setLang(lang)` → 存储到 localStorage + 刷新页面
- 静态HTML → `applyStaticText()` 初始化时一次性替换
- 动态渲染（项目卡片/详情/技能）→ 模板字符串中直接调 `t()`

## 涉及的技术决策

| 决策 | 理由 |
|------|------|
| Mermaid.js CDN加载 | 浏览器端实时渲染架构图，无需预生成图片 |
| localStorage 持久化 | 拖拽位置、语言偏好、主题都不需要后端 |
| 纯静态HTML/CSS/JS | 可以直接用浏览器打开，无需npm/构建，发朋友圈直接部署 |
| SVG内联而非图标字体 | 体积更小、改色自由、无字体加载问题 |
| 图片轮播用原生JS定时器 | 避免引入Swiper等额外依赖 |
| i18n用刷新重载而非动态替换 | 代码简洁，避免DOM同步bug |
